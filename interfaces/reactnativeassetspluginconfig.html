Optional assetsOutputPath

assetsOutputPath: undefined | string
  • Defined in src/webpack/plugins/ReactNativeAssetsPlugin/ReactNativeAssetsPlugin.ts:25

Directory where all assets (eg: images, video, audio) should be saved. If not provided, all assets will be saved in the same directory as outputPath.

Optional bundleToFile

bundleToFile: undefined | boolean
  • Defined in src/webpack/plugins/ReactNativeAssetsPlugin/ReactNativeAssetsPlugin.ts:32

Whether the build produces static bundle saved to file or the bundle will be updated multiple times and resides in memory.

When development server is running, bundleToFile should be set to false.

context

context: string
  • Defined in src/webpack/plugins/ReactNativeAssetsPlugin/ReactNativeAssetsPlugin.ts:15

Context in which all resolution happens. Usually it's project root directory.

Optional outputPath

outputPath: undefined | string
  • Defined in src/webpack/plugins/ReactNativeAssetsPlugin/ReactNativeAssetsPlugin.ts:20

Bundle output path - directory where built bundle will be saved. If not provided it will be inferred from Webpack configuration.

platform

platform: string
  • Defined in src/webpack/plugins/ReactNativeAssetsPlugin/ReactNativeAssetResolver.ts:17

Target application platform.

Optional test

test: undefined | RegExp
  • Defined in src/webpack/plugins/ReactNativeAssetsPlugin/ReactNativeAssetResolver.ts:15

Override default test RegExp. If the asset matches the test RegExp, it will be process by the custom React Native asset resolver. Otherwise, the resolution will process normally and the asset will be handled by Webpack.
